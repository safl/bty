{# Two tiers of config (see docs/concepts "general vs per-policy"): the boot policy is the MODE selector; the "Machine configuration" section below holds the machine's GENERAL properties (image, target disk, sanboot drive, hostname). Each policy READS a subset of those. The inline script DIMS + annotates the fields the selected policy ignores rather than hiding them, so the general/consumed-by split stays visible to the operator. #}
sanboot: iPXE boots the local disk itself (drive below; defaults to the first disk), falling back to the firmware order if it can't. The default for an assigned box. bty-flash-always: PXE boots chain into the live env so the box re-flashes the assigned image every time, then sanboots the just-flashed disk before the next reflash (per-job CI cadence). bty-flash-once: same as bty-flash-always for the next PXE boot, then auto-flips to sanboot after the live env signals completion. bty-tui: PXE boots chain into the live env and bty on tty1 drops the operator into the wizard to pick an image by hand. bty-inventory: each boot chains the live env just to re-report this box's disks, then sanboots the disk - so swapped hardware is picked up. The auto-discovery default for unknown MACs; it's how the Target disk dropdown below gets populated.
